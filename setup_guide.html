Hotel Booking System Setup Guide

Full Stack Application (Laravel / React)

1. System Requirements

  • Docker >= 20.10.x
  • Docker Compose >= 2.0.x
  • Git
  • (Optional for local development) PHP >= 8.2
  • (Optional for local development) Node.js >= 18.x

│   └── ...
├── hotel-web/         # Frontend React Application
│   ├── Dockerfile
│   └── ...
└── docker-compose.yml # Main Docker configuration

3. Installation Methods

Method A: Docker Installation (Recommended)

  1. Clone Repository 
    git clone https://github.com/chabbasaad/4CITE.git
cd 4CITE
  2. Environment Setup 
    # Copy backend environment file
cp hotel-api/.env.example hotel-api/.env
  3. Start Docker Environment 
    # Build and start all services
docker-compose up -d --build
  4. Initialize Backend 
    # Generate application key
docker-compose exec backend php artisan key:generate

# Run database migrations and seeders
docker-compose exec backend php artisan migrate --seed

4. Docker Services

  • Backend (Laravel): PHP-FPM service running Laravel application
  • Nginx: Web server for the backend API
  • MySQL: Database server
  • Frontend (React): Vite development server for React application

5. Access Points

  • Frontend Application: http://localhost:5173
  • Backend API: http://localhost:8000
  • API Documentation: http://localhost:8000/docs/api
  • MySQL: localhost:3306

6. Docker Management Commands

# Start all services
docker-compose up -d

# Stop all services
docker-compose down

# View logs of all services
docker-compose logs -f

# View logs of specific service
docker-compose logs -f backend
docker-compose logs -f frontend
docker-compose logs -f nginx

# Rebuild services
docker-compose up -d --build

# Backend Commands
docker-compose exec backend php artisan list
docker-compose exec backend php artisan migrate
docker-compose exec backend composer install

# Frontend Commands
docker-compose exec frontend npm install
docker-compose exec frontend npm run build

7. Testing Guide

Backend Testing

# Run all tests
docker-compose exec backend php artisan test

# Run specific test suites
docker-compose exec backend php artisan test --testsuite=Unit
docker-compose exec backend php artisan test --testsuite=Feature

# Run specific test class
docker-compose exec backend php artisan test --filter=AuthenticationTest

Frontend Testing

# Run tests
docker-compose exec frontend npm test

# Run tests with coverage
docker-compose exec frontend npm test -- --coverage

# Run tests in watch mode
docker-compose exec frontend npm test -- --watch

8. Troubleshooting

  • Permission Issues: 
    chmod -R 777 hotel-api/storage hotel-api/bootstrap/cache
  • Container Access: 
    # Access backend container
docker-compose exec backend bash

# Access frontend container
docker-compose exec frontend sh
  • Reset Environment: 
    # Remove all containers and volumes
docker-compose down -v

# Rebuild from scratch
docker-compose up -d --build

API Documentation with Scramble

The project uses Scramble to automatically generate OpenAPI (Swagger) documentation for all API endpoints.

Features

  • Automatic OpenAPI 3.1.0 specification generation
  • Interactive API documentation UI using Stoplight Elements
  • Real-time documentation updates based on code changes
  • Authentication endpoints documentation
  • Request/Response schema documentation

Accessing Documentation

  1. Local Environment: 
    http://localhost:8000/docs/api
  2. Docker Environment: 
    http://localhost:8000/docs/api

Documentation Features

  • Interactive endpoint testing
  • Request/Response examples
  • Authentication flow documentation
  • Schema definitions for all models
  • Detailed error responses
